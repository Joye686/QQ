📊 Power BI 设计方案 - 客户流向分析

1. 项目概述

分析客户选择与实际发出商品的流向差异，识别业务中的偏差和优化机会。

分析目标

  • 可视化客户选择 vs 实际发出的流向对比
  • 识别主要偏差来源
  • 支持业务决策优化

2. 数据源

数据来源

  • 客户选择记录：客户下单时的选择数据（实时更新）
  • 实际发出记录：仓库实际发货数据（实时更新）
  • 商品主数据：商品分类、规格等信息（每日更新）

关键字段

客户 ID、订单号、商品 SKU、选择数量、发出数量、时间戳、仓库/发货地

3. 可视化模块

3.1 桑基图 (Sankey Diagram)

  • 左侧节点：客户选择的商品分类
  • 右侧节点：实际发出的商品分类
  • 连线宽度：代表流量大小
  • 颜色编码：区分商品类别

3.2 辅助图表

  • 偏差分析柱状图：展示各品类的选择 vs 发出差异
  • 时间趋势折线图：按日/周/月展示偏差变化趋势
  • 客户分布地图：地理位置维度的流向分布

4. 交互功能

筛选器

  • 时间范围：日期选择器（默认：最近 30 天）
  • 客户分类：下拉多选（默认：全部）
  • 商品品类：下拉多选（默认：全部）
  • 发货仓库：下拉单选（默认：全部）

钻取功能

  • 从品类钻取到 SKU
  • 从月份钻取到日期
  • 从总览钻取到明细

工具提示

  • 鼠标悬停显示详细数据
  • 显示偏差百分比和绝对值

5. 技术实现

Power BI 组件

  • 桑基图自定义视觉对象：流向可视化
  • 标准柱状图：偏差对比
  • 折线图：趋势分析
  • 地图视觉对象：地理分布

数据模型

客户选择表 + 实际发出表 → 关联商品主数据

DAX 关键度量

  • 偏差率 = (实际发出 - 客户选择) / 客户选择
  • 偏差量 = 实际发出 - 客户选择

6. 部署与分享

发布流程

  1. Power BI Desktop 开发
  2. 发布到 Power BI Service
  3. 配置数据刷新计划
  4. 分享给相关人员

访问权限

  • 业务分析师：编辑 + 查看
  • 业务经理：查看 + 导出
  • 执行人员：查看

数据刷新

  • 频率：每日自动刷新
  • 时间：凌晨 2:00
  • 失败通知：邮件提醒
